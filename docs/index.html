The difference

One theme, four layers.

Most tools stop at the prompt. Posh Palette coordinates every layer that decides how your shell looks — from a single palette, applied at once.

layer 01

Color scheme

Windows Terminal

Background, foreground, and the 16 ANSI colors. Hot-reloads the instant it's applied.

layer 02

Typed commands

PSReadLine

Commands, parameters, strings, and errors colored as you type them.

layer 03

Command output

$PSStyle

Directory listings, error text, and table headers in the output stream.

layer 04

Prompt

oh-my-posh

Your prompt theme, kept in step with everything around it.

Two ways in

Browse a look, or build one.

Start from a finished theme, or assemble one from the catalogs — both show a live preview before anything touches your config.

simple mode

Scroll and pick

Arrow through a list of complete themes with a live preview beside it. Press Enter — the terminal, your prompt, and your colors all change at once.

detail mode

Compose layer by layer

Keep your prompt, swap just the color scheme. Mix a scheme, a palette, a prompt, and a font from the catalogs — and watch each change in the preview before you commit.

Open by design

A theme is one small file.

It doesn't hold colors — it points at entries in per-layer catalogs. That's what lets you swap a single layer, and what makes every layer its own contribution.

  • schemes/16 ANSI colors + background, foreground, cursor
  • palettes/PSReadLine input + $PSStyle output colors
  • prompts/an oh-my-posh theme reference
  • fonts.jsonthe nerd fonts you can pick from

Add your own — one JSON file, one pull request.

themes/tokyo-night.json { "name": "Tokyo Night", "scheme": "tokyo-night", "palette": "tokyo-night", "prompt": "tokyonight-storm", "font": "jetbrains" }
